> **상태**: 기획 완료, 개발 미착수  
> **작성일**: 2026-06-26  
> **작성자**: 보기 (심플라인)

---

## 1. 개요

### 목적
DXF/STEP 도면을 업로드하면 재료비·가공비·후처리·관리비를 자동 산출하는 사내 판금 견적 웹앱.

### 배경
현재 견적은 수작업으로 산정 → 담당자 의존도 높고, 단가 기준이 문서화되어 있지 않음.  
도면에서 치수·형상 정보를 자동 추출하면 견적 시간을 단축하고 기준을 일관화할 수 있음.

### 성공 기준
- 도면 업로드 후 3분 이내 견적가 산출
- 수작업 견적 대비 오차 ±10% 이내 (단가 정립 후 검증)
- 단가표를 코드 수정 없이 웹 UI에서 편집 가능

---

## 2. 사용자 & 시나리오

### 주 사용자
심플라인 내부 견적 담당자 (설계 실무자 포함)

### 핵심 시나리오
1. 고객 문의 도면(DXF 또는 STEP)을 업로드
2. 재질·두께·수량·후처리 종류를 선택 (파싱으로 자동 입력, 필요 시 수정)
3. 절곡비 계산 방식 선택 (P4 패널벤더 / 일반 절곡기)
4. 견적 산출 결과를 화면에서 검토
5. PDF로 다운로드해 내부 보관 또는 고객 전달

## 3. 핵심 기능 요구사항

### F1. 도면 파일 업로드 & 파싱
| 항목 | DXF | STEP |
|------|-----|------|
| 전개면적 | 외곽 폴리라인 면적 | - |
| 순중량 기반 부피 | 전개면적 × 두께 × 비중 (근사) | 솔리드 부피 × 비중 (정확) |
| 절단장 | 외곽 + 구멍 둘레 합산 | - (미지원) |
| 구멍 수 | 원/호 엔티티 카운트 | - (미지원) |
| 절곡 횟수 | 절곡선 레이어 카운트 | 피처 수 (가능 시) |
| 표면적 | 전개면적 × 2 + 둘레 × 두께 (근사) | 솔리드 표면적 (정확) |

**파싱 실패 시**: 각 항목을 수동으로 직접 입력할 수 있는 폼 제공.

### F2. 재질·두께·후처리 선택
- **재질**: SPCC, SUS304, SUS316, AL5052, AL6061 (추후 확장)
- **두께**: 0.8 / 1.0 / 1.2 / 1.5 / 2.0 / 2.3 / 3.0 / 4.5 mm (추후 확장)
- **후처리**: 없음 / 분체도장 / 도장(일반) / 도금 (선택지)
- **절곡 방식**: P4 패널벤더 / 일반 절곡기 (사용자 직접 선택)
- **수량**: 정수 입력

### F3. 원가 산정 엔진

#### 재료비
```
재료비 = 순중량 × kg단가[재질]
순중량 (DXF) = 전개면적(㎡) × 두께(m) × 비중(kg/m³)
순중량 (STEP) = 부피(m³) × 비중(kg/m³)
```

#### 절단비
```
절단비 = 절단장(m) × 절단m단가[두께] + 구멍수 × 피어싱단가[두께]
```

#### 절곡비
```
절곡비 = 절곡횟수 × 회당단가[P4 or 일반]
```

#### 도장/후처리비
```
표면적 (DXF) = 전개면적 × 2 + 외곽둘레 × 두께
표면적 (STEP) = 솔리드 표면적 (직접 추출)

후처리비 = 표면적(㎡) × 후처리㎡단가[후처리종류]
```

#### 관리비·마진
```
소계 = 재료비 + 절단비 + 절곡비 + 후처리비
관리비 = 소계 × 관리비율
마진 = 소계 × 마진율
합계 = 소계 + 관리비 + 마진
```

#### 수량 반영
```
최종 견적가 = 합계 × 수량
```
> 수량 할인 로직은 MVP 이후 검토.

### F4. 단가 관리 (관리자 UI)
- 별도 관리자 페이지 (`/admin/pricing`)
- 항목별 단가를 테이블 형태로 조회·수정·저장
- 단가 변경 이력 저장 (날짜 + 변경 전/후 값)

**단가 테이블 구조**

| 테이블 | 키 | 값 |
|--------|----|----|
| material_price | 재질 | kg단가 (원) |
| cut_price | 두께(mm) | m단가 (원/m) |
| pierce_price | 두께(mm) | 피어싱단가 (원/구멍) |
| bend_price | 방식(p4/general) | 회당단가 (원/회) |
| surface_price | 후처리종류 | ㎡단가 (원/㎡) |
| overhead_rate | - | 관리비율 (%) |
| margin_rate | - | 마진율 (%) |

### F5. 견적서 출력
- **화면**: 항목별 계산 내역 + 합계 표 (투명하게 공개)
- **PDF**: 화면 구성 그대로 PDF 다운로드 (`react-pdf` 또는 브라우저 인쇄)
- PDF 파일명: `견적_[부품명]_[날짜].pdf`

## 4. 기술 아키텍처

### 스택
| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | Next.js 15 + TypeScript + Tailwind CSS |
| 백엔드 | Next.js API Routes (Server Actions) |
| DB | SQLite (로컬, Turso 또는 Vercel Postgres 고려) |
| ORM | Prisma 또는 Drizzle |
| DXF 파싱 | `dxf-parser` (npm) |
| STEP 파싱 | `opencascade.js` (WebAssembly) 또는 Python 마이크로서비스 |
| PDF 출력 | `@react-pdf/renderer` 또는 브라우저 인쇄 API |
| 배포 | Vercel (panel-mfg와 동일) |
| 인증 | 없음 (사내 접근 가정) |

### 데이터 흐름
```
도면 업로드
  → (서버) 파싱 → 형상 수치 추출
  → (클라이언트) 재질·두께·후처리·절곡방식 선택·수정
  → (서버) 단가 DB 조회 + 원가 공식 계산
  → 결과 표시 + PDF 다운로드
```

### STEP 파싱 결정 포인트 (미결)
`opencascade.js`는 번들 크기가 크고 초기화가 느림.  
대안: Python (pythonocc / CadQuery)으로 별도 마이크로서비스 구성.  
→ **MVP는 DXF 우선 구현, STEP는 2단계에서 결정.**

## 8. DXF 레이어 규칙 (2개 도면 교차 검증 완료 — 2026-06-27)

### 인코딩
- 파일 인코딩: 한글 (CP949 계열)
- 레이어명: 한글
- MTEXT 내 텍스트: `\U+XXXX` 유니코드 이스케이프 → 파서에서 디코딩 필요

### 레이어 용도표

| 레이어명 | 용도 | 엔티티 | 파싱 방법 |
|----------|------|--------|-----------|
| `외형선` | 부품 윤곽 + 구멍 | LINE / ARC / LWPOLYLINE / CIRCLE | 절단장·구멍 추출 |
| `굽힘선아래로` | 절곡선 (하향) | LINE | LINE 개수 = 절곡 횟수 |
| `굽힘선위로` | 절곡선 (상향) | LINE | LINE 개수 = 절곡 횟수 |
| `SW_노트` | 제목란 | MTEXT | 재질/두께/중량/수량 추출 |
| `숨은선` | 점선 형상 | - | **무시** |
| `SW_치수` / `SW_중심선` / `SW_테두리` | SolidWorks 자동 생성 | - | **무시** |
| `치수선` / `중심선` / `0` / `Defpoints` | 치수·기준점 | - | **무시** |

### 추출 항목별 방법

#### 재료 메타데이터 (SW_노트 MTEXT)
| 항목 | 파싱 패턴 | 예시 |
|------|-----------|------|
| 재질 | `SPCC\|SUS304\|SUS316\|AL5052\|AL6061` | `SPCC` |
| 두께 | `(\d+\.?\d*)\s*t\b` | `1 t` → 1.0 mm |
| 중량 | `(\d+\.?\d*)\s*kg` | `2.75 kg` |
| 수량 | `(\d+)\s*(ea\|pcs?)` | `1 ea / Set` |

#### 절단장 · 구멍 수 (외형선 레이어)
```
절단장 = Σ LINE 길이 + Σ ARC 호길이 + Σ LWPOLYLINE 길이 + Σ CIRCLE 둘레
구멍 수 = 외형선 CIRCLE 개수
```

#### 절곡 횟수
```
절곡 횟수 = 굽힘선아래로 LINE 수 + 굽힘선위로 LINE 수
```
- LINE 1개 = 절곡 1회 (병합 없이 단순 카운트)
- 위로/아래로 방향 각각 별도 기록, 합산이 총 절곡 횟수

#### 순중량 산정 (2단계 폴백)
1. **1순위**: SW_노트에서 중량(kg) 직접 읽기
2. **2순위**: 외형선 LWPOLYLINE 면적 × 두께 × 비중으로 역산

| 재질 | 비중 (g/cm³) |
|------|-------------|
| SPCC / SECC / SGCC / SS400 | 7.85 |
| SUS304 / SUS316 | 7.93 |
| AL5052 / AL6061 | 2.71 |

### 비표준 도면 처리 원칙
- 파서는 항목별 `추출 성공 / 미검출` 을 명시적으로 리포트
- 미검출 값은 `0`이 아닌 `null`로 구분 (0과 미검출은 의미가 다름)
- 표준 레이어(`외형선` / `굽힘선아래로` / `굽힘선위로` / `SW_노트`)가 하나도 없으면 → "비표준 도면" 판정, 오류 중단 아님
- 견적 화면에서 자동 추출값은 '기본 채움'으로 표시, 모든 항목 수정 가능
- 파서가 아무것도 읽지 못해도 전 항목 수동 입력으로 견적 완성 가능

### 가공 종류 (TAP / 버링 / 엠보) — MVP 제외
- 특수 가공은 도면 내 심볼로만 표기, 범례표(X ≈ 4570~4830 좌표 영역)에만 텍스트
- MVP: 외형선 CIRCLE을 단순 구멍 수로만 카운트, 가공 종류별 수량은 견적 화면에서 수동 입력
- 심볼 자동 인식은 v2 항목
