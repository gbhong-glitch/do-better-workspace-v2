## 지시 1 — DXF 화면 렌더링

```
DXF 파일 1장을 웹 화면에 렌더링하는 기능을 만들어줘.
- 레이어별로 선/원/호/폴리라인/텍스트를 그대로 그림 (SVG 또는 Canvas)
- 줌/팬 가능
- Next.js + TypeScript
- 단가·견적·인식은 아직 붙이지 마. 도면이 화면에 제대로 보이는지만.
```

## 지시 2 — 드래그로 영역 선택

```
렌더링된 도면 위에서 마우스 드래그로 사각 영역(박스)을 그리는 기능 추가.
- 드래그하면 반투명 박스가 그려짐
- 박스 좌표(x1,y1,x2,y2)를 상태로 저장
- 박스 여러 개 생성 가능 (조립체마다 1개)
- 각 박스에 이름 지정 (조립체-01, -02 ...)
```

## 지시 3 — 영역 안 레이어 인식

```
parse_simpleline.py 의 인식 로직을 TypeScript로 이식하되,
X_SPLIT 같은 위치 하드코딩은 제거하고 '드래그 박스 좌표'를 입력으로 받게 해줘.
- 입력: 박스 좌표 + DXF 엔티티
- 박스 안에서 레이어 기반으로 인식:
  · 재단방식 (레이저/복합기/NCT/절단 레이어 텍스트) = 세부부품
  · 절곡 (굽힘선아래로/위로 LINE) = 세부부품별로 분리 (합계 아님)
  · 특수가공 (블록 이름)
  · 재질/두께/수량 (SW_노트)
- recognition-rules.md 의 규칙을 따를 것
- 결과: 박스(조립체)별 → 세부부품 목록(재단방식+절곡+재질)
```

## 지시 4 — 인식 결과 표시

```
드래그한 각 조립체 박스별로 인식 결과를 표로 표시.
- 조립체별: 세부부품 목록 (재단방식 / 절곡 수 / 재질·두께·수량)
- 파이프 절단 표 별도 표시
- 특수가공 종류·개수 표시
- 여기까지가 1차 목표: 단가 없이 '인식이 정확한가'만 확인
```

## 지시 5 — 실제 도면 검증

```
실제 도면 여러 장으로 지시1~4를 테스트.
- 깨지는 부분 기록 (배치 다름/레이어 누락/인식 오류)
- 자동으로 안 되는 건 드래그/수동입력으로 보완하는 UI 추가
```

## 지시 6 — 절곡 단가 연결 (인식 안정화 후)

```
bending_pricing.json + bending_calc.py 의 절곡 단가를 연결.
- 세부부품별 절곡 길이 → 구간별 단가 적용
- 헤밍/V컷/포밍/비딩은 수동 입력
```
