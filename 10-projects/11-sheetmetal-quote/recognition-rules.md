# 심플라인 도면 인식 규칙 (검증 완료)

예제.dxf 분석으로 확정한 도면 구조와 인식 규칙. 통합 파서 `parse_simpleline.py`에 구현됨.

## 도면 구조

도면 1장에 여러 **조립체(용접도)**가 좌우로 배치되고, 각 조립체 아래에
그것을 구성하는 **세부부품**들이 전개도로 펼쳐진다.

```
도면 1장
 ├─ 용접도(조립체) 좌 = 부품 -01   (X < 8000)
 │    └─ 세부부품 N개
 └─ 용접도(조립체) 우 = 부품 -02   (X > 8000)
      └─ 세부부품 M개
```

용접도가 최상위 단위 = 조립체 = 부품번호(-01/-02)의 기준.

## 인식 규칙 (레이어 기반)

심플라인 도면은 레이어 이름에 정보가 담겨 있어, 형상 분석 없이
레이어·텍스트만 읽어 대부분 자동 구분된다.

| 항목 | 인식 근거 | 비고 |
|---|---|---|
| 조립체(부품) 구분 | `용접도`·`자품번` 레이어 텍스트의 X좌표 | X<8000=-01, X>8000=-02 |
| 세부부품 | 재단방식 레이어 텍스트 1개 = 세부부품 1개 | 레이저/복합기/NCT/절단 |
| 재단방식 | 위 재단방식 레이어 이름 | |
| 절곡 | `굽힘선아래로`·`굽힘선위로` LINE 개수 | 전부 카운트(병합 없음) |
| 절곡-부품 배정 | 같은 부품(side) 내 Y최근접 재단라벨 | |
| 전개도(재단길이) | **굽힘선이 포함된 영역** = 전개도 | 삼각도·등각도엔 굽힘선 없음 |
| 재단 길이 | 전개도 영역 내 `외형선` 둘레 합 | 외곽+내부홀 |
| 파이프 절단 | `절단` 라벨 근처 표(품번/규격/재질/수량/길이/각도) | 텍스트 표, TABLE객체 아님 |
| 재질·두께·수량 | `SW_노트` 텍스트, 세부부품 Y근접 | 부품마다 다름 |
| 특수가공 | 블록(INSERT) 이름 패턴 | TAP/버링/EM/BUR/자석 |

## 무시 대상

- `SW_*`(SW_치수/SW_중심선/SW_노트 일부)·`CENTERMARK` 블록 — SW 자동생성
- `A$C...`·`*` 블록 — AutoCAD 익명 블록
- `FORMAT` — 도면 양식
- `숨은선`·`나사산`·`중심선` — 참고선 (재단/절곡 아님)

## 검증 결과 (예제.dxf)

- 조립체 2개: KTFO-310-02-TB-1100-01-01 / -02
- 세부부품 13개: 부품01(복합기5·레이저1·절단1=7개), 부품02(레이저6)
- 절곡 합계 76곡
- 파이프 절단 2종: 40×20×1.4 HR (824mm×2, 2373mm×2)
- 특수가공: 버링탭 12개

## 미해결 / 보완 예정

1. **굽힘선 부품별 경계 정밀도** — 부품 경계에서 굽힘선이 인접
   세부부품으로 새는 경우가 있어 절곡 수·재단 길이가 일부 부정확.
   여러 도면으로 검증하며 클러스터링 개선 필요.
2. **양식 파라미터 하드코딩** — X_SPLIT=8000 등이 이 도면 기준.
   다른 규격 도면은 양식(FORMAT)에서 자동 추출하도록 개선 가능.
3. **품명 매칭** — 일부 세부부품 품명이 빈 값. 노트 위치 규칙 추가 확인 필요.
4. **용접비·재단길이 단가** — 미정. 재단방식·절곡 단가는 bending_pricing.json 참고.
